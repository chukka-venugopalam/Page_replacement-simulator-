Algorithm Comparison: FIFO vs LRU vs Optimal

Concepts & Time Complexity

FIFO (First-In, First-Out)

Concept: Replaces the oldest page in memory. It treats page requests like a queue.

Pros/Cons: Very simple to code, but suffers from Belady's Anomaly (more frames can sometimes cause more faults).

Time Complexity: O(1) per step (just moving a pointer).

LRU (Least Recently Used)

Concept: Replaces the page that has not been used for the longest time in the past.

Pros/Cons: Excellent real-world performance. Doesn't suffer from Belady's Anomaly. Harder to implement efficiently in hardware.

Time Complexity: O(N) per step in our array, but can be O(1) with a Hash Map + Doubly Linked List.

OPT (Optimal)

Concept: Looks into the future and replaces the page that will not be used for the longest time.

Pros/Cons: Guarantees the lowest possible page faults. However, it is impossible to implement in real life since OS cannot predict the future.

Time Complexity: O(N) per step (requires looking ahead in the array).
